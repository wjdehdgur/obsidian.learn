## ✅ MOM (Message Oriented Middleware)

### 🔤 용어 해석

- **Message**: 메시지, 데이터 단위
    
- **Oriented**: ~지향의
    
- **Middleware**: 미들웨어, 시스템 간 중간 연결 소프트웨어  
    → 즉, **메시지 기반의 시스템 연결을 위한 미들웨어**
    

---

### 🧩 MOM이란?

|항목|설명|
|---|---|
|**정의**|서로 다른 시스템(또는 모듈)이 **메시지를 교환할 수 있도록 중개하는 미들웨어**|
|**기능**|데이터 송수신을 **비동기적(Asynchronous)** 또는 **비접속형 방식으로 전달**|
|**역할**|시스템 간 통신에서 **느슨한 결합, 안정성, 유연성**을 보장|

---

### 📬 MOM의 특징

| 특징          | 설명                                                          |
| ----------- | ----------------------------------------------------------- |
| **비동기 통신**  | 송신자와 수신자가 **동시에 연결되지 않아도 됨**                                |
| **큐 기반 전송** | 메시지를 임시 저장하는 **[[Message Queue\|메시지 큐(message queue)]]** 사용 |
| **플랫폼 독립성** | OS, 언어, 환경이 달라도 메시지로 연결 가능                                  |
| **느슨한 결합**  | 시스템 간 의존도 낮음 → 확장성 좋음                                       |

---

### 🔧 구조 구성

```
[발신자] → (메시지 생성) → [MOM (메시지 큐)] → (전달) → [수신자]
```

- 발신자는 메시지를 큐에 넣고 종료
    
- 수신자는 자신이 처리 가능한 시점에 메시지를 꺼내 처리
    

---

### 📌 주요 구현 예

|제품|설명|
|---|---|
|**RabbitMQ**|오픈소스 기반 메시지 브로커|
|**Apache Kafka**|대용량 실시간 메시지 스트리밍에 적합|
|**IBM MQ**|기업용 메시징 솔루션|
|**ActiveMQ**|Java 기반 메시징 미들웨어|

---

### 🎯 MOM vs [[Remote Procedure Call|RPC]] 비교

|구분|MOM|RPC|
|---|---|---|
|연결 방식|비동기, 비접속|동기, 연결 기반|
|의존성|낮음 (느슨한 결합)|높음 (강한 결합)|
|대표 방식|큐 기반 메시징|함수 호출 기반 통신|

---

### 🧠 정리

|항목|내용|
|---|---|
|핵심 개념|**메시지 큐를 통해 비동기적으로 시스템을 연결**하는 미들웨어|
|활용 사례|**분산 시스템, IoT, 이벤트 기반 아키텍처, 비동기 처리 시스템**|
|시험 포인트|"비동기 메시지 기반 미들웨어", "느슨한 결합", "큐 사용" 기억하기|
